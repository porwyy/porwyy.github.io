nosql学习笔记（一）

特点：

  • 易扩展：去掉关系数据库的关系型特性（最后附上关系型数据库特性）。数据之间无关系，这样就非常容易扩展，也使得在架构的层面上带来了可扩展的能力。
  • 读写高性能：nosql良好的Cache特性，使得其具有很好的读写性能
  • 多样灵活的数据模式：NoSQL无需事先为要存储的数据建立字段，随时可以存储自定义的数据格式。

RDBMS vs NoSQL

RDBMS

  • 高度组织化结构化数据
  • 结构化查询语言（SQL）
  • 数据和关系都存储在单独的表中。
  • 数据操纵语言，数据定义语言
  • 严格的一致性
  • 基础事务

NoSQL

  • 代表着不仅仅是SQL
  • 没有声明性查询语言
  • 没有预定义的模式
    -键 - 值对存储，列存储，文档存储，图形数据库
  • 最终一致性，而非ACID属性
  • 非结构化和不可预知的数据
  • CAP定理
  • 高性能，高可用性和可伸缩性

关系型数据库

特性：(ACID)

  1. 一致性：事务处理前后，保证数据在整体上数量一致，如对银行账户而言，无论如何转账，存款总金额不变；
  2. 原子性：即一个事务要么做完，要么不做；
  3. 独立性：数据库为每一个用户开启的事务，不受其他事务干扰。例如若一个正在被访问的数据，也在被另一用户修改，则只要修改后的数据未提交，被访问的数据就是原数据；
  4. 持久性：某一事务一旦提交完成，对数据库产生的改变是持久的，不再改变。
